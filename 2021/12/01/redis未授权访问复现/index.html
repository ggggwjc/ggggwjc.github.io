redis未授权访问复现

2021-12-01

前面的环境配置就不做记录了。

靶机：centos7 192.168.0.131

攻击机：kali 192.168.0.39

1.利用redis写webshell

先通过 redis-cli -h 192.168.0.131 连接redis

查看靶机

在桌面生成shell.php

2.利用”公私钥”认证获取root权限

在靶机中执行 mkdir /root/.ssh 命令，创建ssh公钥存放目录（靶机是作为ssh服务器使用的）

在kali中生成ssh公私钥，密码为空

进入ssh目录把公钥保存到1.txt

将保存ssh的公钥1.txt写入redis

登入redis，使用config set dir 获得redis的备份路径

更改redis备份路径为ssh公钥存放目录(/root/.ssh)

设置上传公钥的备份文件名字为authorized_keys并保存

使用 ssh -i id_rsa root@192.168.0.104 连接
